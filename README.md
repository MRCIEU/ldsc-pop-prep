# Generate LDscores for ldsc regression

This repository contains a nextflow pipeline and data to generate LDscores for LDscore regression, as described in the [wiki](https://github.com/bulik/ldsc/wiki/LD-Score-Estimation-Tutorial) of the [original ldsc software](https://github.com/bulik/ldsc).

The LDscores themselves are generated using [a newer version](https://github.com/CBIIT/ldsc) of the ldsc software, which has been translated to python3.

A set of scores will be generated for each of the following 1000 Genomes populations:

    1) AFR
    2) AMR
    3) EAS
    4) EUR
    5) SAS
    6) ALL (samples included from all of the above populations)

The pipeline requires that you download ~14GB of 1000 Genomes data as input. A script to download this data from:

    1) https://ftp.1000genomes.ebi.ac.uk (vcf files), and
    2) https://mathgen.stats.ox.ac.uk (genetic maps)

exists in this repository in `data/download_data.sh`. Run this shell script before running the nextflow pipeline.

88MB of precomputed information about ~1.2 million HapMap3 SNPs is already present in `data/hm3/`, and is also required to run the pipeline. These data consist of the list of snps (labelled by RSID) (`data/hm3/w_hm3.snplist`), and per-chromosome tables linking snps (RSIDs) to their genomic location, in hg37 coordinates. These data were generated by the script `scripts/get_RSID_info.R`.

### Running the pipeline

```
nextflow run main.nf
```

Optionally, you can set the sample size to (randomly) subset the populations to, prior to calculating LDscores, and specify the name of the directory to write the results to:

```
nextflow run main.nf --n 100 --results_dir output
```

If you don't set these options, their defaults are n=300 and results_dir=results

### Output

The (per-chromosome) LDscores are written to 

```
{results_dir}/
    ldscores/
        AFR/
        ALL/
        AMR/
        EAS/
        EUR/
        SAS/
```

And the sample lists retained after subsetting the data to `n` individuals for each population are written to

```
{results_dir}/sample_lists/{POP}.samples.tsv
```

### Notes

The pipeline runs its processes in the docker containers specified in `nextflow.config`.

Related samples (as defined in `data/1000G_vcf/20140625_related_individuals.txt`) are removed prior to choosing samples for inclusion in the LD calculations.
